How the Integration Works

The system relies on a three-tier architecture: Execution (Playwright + Custom Reporter), Publication (GitHub Actions), and Visualization (Chrome Extension v1.2).

  • Passed/Failed Badge: The icon itself now displays live counts (e.g., "42/1").
  • 1-Minute Polling: Stay in sync with your CI/CD at near real-time speeds.
  • Smart Error Detection: Automatically flags syntax errors or test crashes via a gray badge status.

Step 1: Creating the Summary Reporter

Instead of using the standard, heavy JSON reporter, we'll create a tiny, dedicated script that summarizes our results into a few bytes.

summary-reporter.js
const fs = require('fs');

class SummaryReporter {
  onBegin(config, suite) {
    this.rootSuite = suite;
  }

  onEnd(result) {
    const summary = {
      schemaVersion: 1,
      passed: 0,
      failed: 0,
      flaky: 0,
      total: 0,
      startTime: new Date().toISOString(),
      isSummary: true
    };

    if (this.rootSuite) {
      for (const test of this.rootSuite.allTests()) {
        const status = test.outcome();
        if (status === 'expected') summary.passed++;
        if (status === 'unexpected') summary.failed++;
        if (status === 'flaky') summary.flaky++;
      }
    }

    summary.total = summary.passed + summary.failed + summary.flaky;
    fs.writeFileSync('test-summary.json', JSON.stringify(summary, null, 2));
    console.log('âœ… Automatic summary created: test-summary.json');
  }
}

module.exports = SummaryReporter;

Step 2: Adding to Playwright Config

Now, register your custom reporter in playwright.config.ts. This allows Playwright to update the summary file automatically at the end of every run.

playwright.config.ts
import { defineConfig } from '@playwright/test';

export default defineConfig({
  // ... other configs
  reporter: [
    ['html'], 
    ['./summary-reporter.js'] // Add your new custom reporter here
  ],
});

Step 3: Automating with GitHub Actions

To keep the extension "live," the test-summary.json file must be updated in your repository after every test run.

Security Note
For private repositories, use the **GitHub Pages** method. This allows the extension to read your test summary without needing sensitive Personal Access Tokens.
.github/workflows/playwright.yml
- name: Update Results Badge
  run: |
    git config user.name "Actions Bot"
    git config user.email "actions@github.com"
    git add test-summary.json
    git commit -m "chore: update test results [skip ci]"
    git push
